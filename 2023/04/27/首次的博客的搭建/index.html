1、Node.js安装及环境配置之Windows篇 - 刘奇云 - 博客园 (cnblogs.com)

常用的npm(Node.js的包管理工具)命令
  1. npm install hexo-cli -g（g代表global）
  2. npm list -g 查询global的位置

2、安装git(以前装过，不在演示了)

3、npm install hexo-cli -g（下载hexo）

4、Github创建仓库

  • 注册登入Github https://github.com/

  • 点击Create a new repository进入新建仓库页面

  • GitHub: Let’s build from here

  • 点击Create a new repository进入新建仓库页面

  • 仓库名输入：用户名.github.io

  • 勾选Public

  • 勾选 Add a README file

  • 拉到下面点击create创建

5、生成SSH Keys

  • 进入任意文件夹，右键空白处然后点Git bush here,输入

  • ssh-keygen -t rsa -C “邮件地址”

  • 然后敲4次Enter⌨️

  • 然后进入C:\Users\用户名，在里面进入.ssh文件

  • 用记事本打开里面的id_rsa.pub,全选复制里面的代码

  • 复制到Github中Setting中的新建SHH中

Bash是一种Unix和Linux操作系统的命令行解释器

6、本地生成博客

  • 在喜欢位置新建文件Blog，然后进入文件夹

  • 右键空白处然后点Git bash here，输入hexo init

  • 在输入hexo g生成静态页面

hexo g 是 Hexo 命令行工具中的一个命令，用于生成静态页面。

具体来说，hexo g 命令会根据你在 Hexo 博客中编写的 Markdown 文章和配置文件，生成一个静态网站。这个网站包括 HTML、CSS、JavaScript 文件和图片等资源，这些文件都可以直接部署到服务器上，让你的博客网站上线运行。

在生成静态页面之前，Hexo 会先读取 _config.yml 配置文件和主题配置文件，然后通过解析 Markdown 文件和渲染模板来生成 HTML 页面。生成的页面会保存在 public 目录中。

总之，hexo g 命令是 Hexo 博客网站构建的核心命令之一，没有它就无法生成静态页面，也就无法让你的博客网站上线运

  • 再输入hexo s启动服务器
  • 编辑_config.yml 更改deploy后面的内容
  • 再Blog文件夹下面运行CMD执行

  • 重新执行hexo g、hexo s

  • hexo new 文章标题 用于存放的文件夹在R:\Install\Java\source_posts
